# Java Interview Questions for 4-Year Experienced Developers

This section contains comprehensive Java coding interview questions designed specifically for developers with around 4 years of experience. Each question includes multiple solution approaches, detailed explanations, and test cases.

## 📚 Categories

### 🗂️ Collections (`collections/`)
- **DuplicateElementsInArray.java** - Find duplicates using HashSet, Streams, and frequency mapping
- **CustomHashMap.java** - Complete HashMap implementation with collision handling and resizing
- **LinkedListImplementation.java** - Generic singly linked list with advanced operations (reverse, cycle detection, merge)

### 🧮 Algorithms (`algorithms/`)
- **TwoSum.java** - Classic two-sum problem with multiple approaches (brute force, HashMap, two pointers)
- **LRUCache.java** - Least Recently Used cache implementation using HashMap + Doubly Linked List
- **SlidingWindowMaximum.java** - Find maximum in sliding windows using Deque, TreeMap, and other approaches

### 🌊 Streams (`streams/`)
- **EmployeeStreamOperations.java** - Complex stream operations on employee data (grouping, filtering, sorting)
- **StreamParallelProcessing.java** - Parallel streams, performance considerations, and thread safety

### 🔤 Generics (`generics/`)
- **GenericWildcards.java** - Comprehensive guide to wildcards, PECS principle, and type safety

### 🔄 Concurrency (`concurrency/`)
- **ProducerConsumerPattern.java** - Multiple implementations using BlockingQueue, wait/notify, Semaphores, and CompletableFuture

## 🎯 What Each Question Tests

### For 4-Year Experience Level:
- **Data Structure Design**: Custom implementations of HashMap, LinkedList, LRU Cache
- **Algorithm Optimization**: Time/space complexity analysis, multiple solution approaches
- **Java 8+ Features**: Advanced Stream API usage, parallel processing, functional interfaces
- **Concurrent Programming**: Thread safety, synchronization mechanisms, modern concurrency utilities
- **Generic Programming**: Type system understanding, wildcards, type safety
- **Problem Solving**: Real-world scenarios, edge case handling, performance considerations

## 🚀 How to Use

1. **Choose a question** based on the topic you want to practice
2. **Read the problem statement** and examples carefully
3. **Try solving it yourself** before looking at the solutions
4. **Study multiple approaches** - each solution teaches different concepts
5. **Run the tests** by removing the `@Disabled` annotation
6. **Understand time/space complexity** of each approach

## 🏃‍♂️ Getting Started

To run any question:

1. Navigate to the specific file
2. Remove the `@Disabled` annotation from the test methods
3. Run the tests using your IDE or Maven: `mvn test`
4. Study the implementation and try variations

## 💡 Interview Tips

- **Understand the problem** completely before coding
- **Discuss trade-offs** between different approaches
- **Start with brute force**, then optimize
- **Write clean, readable code** with proper variable names
- **Test with edge cases** (empty input, single element, etc.)
- **Explain your thought process** while coding
- **Know time and space complexity** of your solution
